# Docs And Release Outline

Use these outlines as defaults. Adapt them to the repo instead of copying blindly.

## Public Plugin README Outline

1. Project name
2. Language switch if bilingual
3. What the host plugin or product is
4. What this extension adds
5. Supported providers or engines
6. Installation
7. Quick start
8. Example provider or proxy notes
9. Pricing or comparison links if relevant
10. Compatibility
11. Verification commands
12. License

## GitHub Pages Landing Page Outline

1. Hero
2. Core features
3. Why use this project
4. Installation
5. Use cases
6. FAQ
7. GitHub call to action
8. Footer

## Usage Guide Outline

1. Where the setting appears in the real UI
2. What each required field means
3. Which fields are optional
4. A known-good example configuration
5. How to test the configuration
6. Common failure cases

## Extension Guide Outline

1. Official provider vs proxy example
2. Authentication model: single token or signed pair
3. Base URL and path conventions
4. Content type or request format options
5. Mapping provider docs to local settings
6. Warnings about vendor-specific assumptions

## Release Checklist Outline

1. Bump version consistently
2. Build the distributable archive
3. Verify the archive installs or imports correctly
4. Update release-facing docs if names or fields changed
5. Draft release notes
6. Attach the archive to the GitHub Release
7. Re-check README and Pages links after publishing
